二叉树的遍历(非递归遍历)

前序遍历

  1. 要注意用栈才能实现纵向遍历，用队列是横向的层序遍历
  2. 子节点入栈的顺序是右->左才能保证出栈的时候是先访问左孩子再访问右孩子

后序遍历

  1. 基于前序遍历来改，子节点入栈顺序是左->右，最后的result顺序是根->右->左
  2. 把result使用stl库函数reverse传两个迭代器进去就是后序遍历

中序遍历

  1. 访问完左孩子节点才能访问根节点和右孩子节点
  2. 前序遍历要访问的节点和要处理的节点是一样的，
  3. 中序遍历不一样，需要定义一个指针访问元素，再通过栈去处理元素

二叉树递归遍历和非递归遍历的区别

  1. 迭代遍历和递归遍历的时间复杂度都是O(n), 空间复杂度O(h)，h是树的高度
  2. 递归遍历深度受到系统调用栈的影响，迭代遍历使用的栈是用户自定义的，更灵活一些

二叉树的层序遍历（广度优先搜索）

  1. 队列每次存储一整层的元素，在弹出元素之前要先记录队列长度将一层元素通过for循环处理完毕
  2. 元素处理的过程是在for循环里面的
  3. 在循环体里面定义path，就能够隔离每一层元素
  4. N叉树的层序遍历，遍历孩子节点可以用迭代器来防止越界，另外，迭代器记得解引用
  5. 二叉树深度和高度的区别，深度是自顶向下，高度是自底向上
    1. 节点深度是根节点到该节点路径长度
    2. 节点高度该节点到最远叶子节点的路径长度
    3. 一整棵树的深度和高度是同一个

二叉树的结构调整

翻转二叉树

  1. 递归法、迭代法（非递归遍历）、层序遍历

对称二叉树

  1. 这题不是层序遍历，只是用了队列这个容器两两检查二叉树对称位置上的节点

最大二叉树

  1. 能不构建新数组就不构建，这题因为可用索引拆分单个数组所以不需要
  2. 切分数组不需要申请额外的空间，有时候指针就可以

合并二叉树

  1. 只有一边有的时候直接返回，不要额外构造

二叉树的遍历

完全二叉树的节点个数

  1. 当作普通二叉树来做 -> 层序遍历统计每一层的宽度
  2. 当作完全二叉树来做（不会也没关系，主要是统计完全二叉树的节点）
    1. 如果当前是满二叉树 -> (2 << depth) + 1
    2. 如果当前不是满二叉树 -> 递归遍历左右孩子节点直到碰到满二叉树（最后一个节点就是一个满二叉树）；countNode(leftDepth) + countNode(rightDepth) + 1最后这个加一是算根节点的
    3. 对于满二叉树来说，第k层有2^k-1个节点（k从1开始计算）

平衡二叉树

  1. 以当前节点为根节点的二叉树已经不是平衡二叉树，返回高度就没有意义了；这个逻辑没有处理好
  2. 记录节点高度的逻辑
    1. 空节点的高度是0
    2. 左右孩子有一个存在，那么对应的值+1；这里很巧的地方是1+max(lhs, rhs)的1+直接跳过了一个变量的定义，还记录了节点的高度
    3. 左右子树高度差超过1，直接返回-1
  3. 深度是自顶向下递增的，所以用前序遍历；高度是自底向上递增的，所以用后序遍历

二叉树的所有路径

  1. string的+只能是string+string，string的push_back()只能作用于char
  2. 前序遍历要把根节点的收集放在结果判断之前，因为最后一个节点也要加入result
  3. 一次递归就要对应一次回溯（指的是左右节点都要进行回溯）

左叶子之和

  1. 前序遍历也可以做，只是所谓的前序遍历在判断根的左孩子节点，所以有点怪

找树左下角的值

  1. 层序遍历最后一个循环就是最底层，只需要记录i == 0的值作为result
  2. 递归遍历需要记录最大深度来判断有没有到达最底层

路径总和

  1. 递归的时候如果想要直接返回，可以用if语句去判断条件是否满足以便直接返回到上一层
  2. 递归的返回值可以是我们想要定义的一个变量
  3. 要注意我们当前的值有没有被减掉

从中序与后序遍历序列构造二叉树

  1. 每次只需要构造根节点，使用递归的返回值去联系根节点和孩子节点（递归到孩子节点的时候他自己就是根节点，所以不用又去new根节点又去new孩子节点
  2. 递归的时候，每层最好只做一件事
  3. 不仅可以加参数，也可以改返回值，返回值是联系相邻两次递归最天然的媒介

二叉树的最近公共祖先

  1. 思路：碰到目标节点的时候直接返回，如果左、右子树不为空，最先满足条件节点就是pq最近的公共祖先
  2. 一定要重新定义节点，不能直接原地修改，否则遍历一次树就变了
  3. 从下往上处理的逻辑可以用回溯来实现，先检查孩子节点再检查根节点可以用后序遍历实现

二叉搜索树相关

二叉搜索树中的搜索

  1. 可以用非递归遍历（前序），碰到值相同的节点直接返回就好了
  2. 这题最好是用二叉搜索树的特性，左子树小右子树大，不需要额外定义一个栈来遍历BST

验证二叉搜索树

  1. 不能单纯判断根节点大于左子树和右子树，因为没有办法比较与叔伯节点之间的关系
  2. 二叉搜索树的中序遍历是个严格递增的有序数组，找中序遍历序列然后再验证

二叉搜索树的最小绝对差

  1. 二叉搜索树的中序遍历是个严格递增的有序数组，针对有序数组找最小绝对差就很简单了
  2. 也可以直接在中序遍历的过程中就找出节点的最小绝对差，因为要记录差所以需要一个容器装结果，一个指针指向前一个节点
  3. 递增数组的差天然就是绝对差，所以这题没必要加abs去求绝对值

二叉搜索树中的众数

  1. 要分离好第一个节点的情况、节点值相同的情况、节点值不相同的情况
  2. 如果出现频率和最大值相同直接push_back，出现频率大于最大值要先clear再push_back；不能合并

二叉搜索树的最近公共祖先

  1. 思路：二叉搜索树的最近公共祖先节点的值介于两者之间，这里不区分pq谁大谁小
  2. 遍历BST过程中第一个满足这个条件的节点就是最近的公共祖先
  3. 为了找到第一个值介于pq两者之间的节点，递归或者迭代法都可以实现

二叉搜索树中的插入操作

  1. 思路：pre，cur遍历二叉搜索树，利用 pre节点来接住新构造的节点newNode
  2. 需要有一个pre节点来接住新构造的节点newNode
  3. pre一开始就要拿到当前节点cur的值，否则cur往下遍历是有可能变成空指针的，pre变成空指针就没有办法变成新节点的父节点
  4. 向二叉搜索树插入元素只需要找到空节点就可以了，不需要调整二叉树的结构
  5. 递归法不是太熟

删除二叉搜索树中的节点

  1. 思路：判断与孩子节点之间的关系，存在以下五种情况
    1. 左右孩子都为空 -> 直接删除
    2. 左孩子为空 -> 右孩子代替当前节点
      1. 这里必须判断左孩子为空的情况，因为两边都非空的情况是在最后处理的
    3. 右孩子为空 -> 左孩子代替当前节点
      1. 这里必须判断右孩子为空的情况，因为两边都非空的情况是在最后处理的
    4. 左右孩子都不为空 -> 左子树接到右子树的最左下节点处后, 右孩子代替当前节点
    5. 没找到要删除的节点, 直接返回
  2. 不用往上判断，只需要判断当前节点是否为要删除的节点
  3. 把左子树接到右子树最左下节点的时候，判断的是cur->left孩子节点，因为要改变指针

修剪二叉搜索树

  1. 思路：判断当前节点，递归寻找满足条件的左右子树
    1. 如果值小于最小值的时候，返回当前节点的右子树（因为左子树只会更小
    2. 如果值大于最大值的时候，返回当前节点的左子树
    3. 最后要记得用root->left接左子树、root->right接右子树
  2. 为什么这个方法不需要在意子树的修剪？ 其实已经做了修剪的工作，返回值就是修剪后的树

将有序数组转换为二叉搜索树

  1. 递归遍历，拆分成左右两份数组+每次只构造根节点
  2. 注意左右两边数组的判空

将二叉搜索树转换为累加树

  1. 这题的关键在于需要按右->中->左的顺序去遍历二叉树
  2. 不属于三种深度优先遍历
  3. 需要一个指针来走向最右边节点，需要一个指针来记录上一个节点的值
  4. 当前节点只需要加上一个节点的值即可，每次只做一个操作
