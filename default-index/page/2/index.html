什么是零拷贝技术？

为什么要有DMA技术?

  1. 早期从磁盘中读取数据是要CPU自己读到寄存器中的，期间CPU没有办法执行其他任务，因为磁盘的读写速度是特别慢的（对比CPU和内存来说），所以CPU自己读是很浪费时间的
  2. DMA控制器就是帮助CPU从磁盘控制器缓冲区里读取数据到内核缓冲区，读完发送中断告知CPU; 在这期间不占用CPU, CPU可以去执行其他任务->提升了系统性能
  3. CPU收到DMA信号以后会将数据从内核缓冲区拷贝到用户空间, 至此, 系统调用返回

传统的文件传输有多糟糕?

  1. DMA控制器将数据从磁盘拷贝到内核空间
  2. CPU将数据从内核空间拷贝到用户空间, 用户才能够使用到这部分数据
  3. CPU将数据从用户空间拷贝到socket, 准备让网卡进行发送
  4. DMA控制器将数据从socket拷贝到网卡的缓冲区中, 进行发送
  5. 为了传输这个数据发生了4次拷贝, 其中CPU参与了2次拷贝, 所以我们在开发的过程中要尽量避免用read, write这样低效率的方式去发送数据或者说发送文件

如何优化文件传输的性能?

关于系统调用 原因
为什么要使用系统调用? 因为用户空间没有操作磁盘或网卡的权限, 只有操作系统内核有权限, 所以需要使用操作系统内核提供的系统调用函数将数据从磁盘或网卡中提取出来
系统调用函数的开销是什么? 系统调用 x 1=上下文切换 x 2, 用户数据要从用户态拷贝到内核态，在内核态处理完数据后又要从内核态拷贝回用户态
如何减少系统调用的次数? 由于内核缓存空间到用户缓存空间以后没有对数据进行再处理，而是直接发给socket缓冲区 -> 用户缓冲区在这里没有必要存在

如何实现零拷贝?

  1. 首先看read函数
    1. read函数需要将内核缓冲区中的数据拷贝到用户缓冲区以后进行读取
    2. 用mmap替换read函数->调用mmap函数以后，DMA会将磁盘中的数据拷贝到内核缓冲区中，然后让应用进程和操作系统共享这个缓冲区；
    3. 应用进程调用write
      1. CPU直接从内核缓冲区拷贝到socket缓冲区 -> 不经过用户缓冲区，减少了1次数据拷贝
    4. 效果：
      1. 因为系统调用的次数还是2次(mmap + write) -> 上下文切换次数还是4次
      2. 用mmap系统调用函数替换read系统调用函数减少了1次数据拷贝
  2. sendfile函数能够在两个文件描述符之间直接传递数据， 避免了内核缓冲区与用户缓冲区之间的数据拷贝， 真正实现了零拷贝
    1. sendfile函数可以代替read + write -> 2次系统调用变成了1次系统调用->减少了2次上下文切换次数
  3. 如果网卡支持SG-DMA（The Scatter-Gather Direct Memory Access） -> 可以进一步减少通过 CPU 把内核缓冲区里的数据拷贝到 socket 缓冲区的过程
  4. 效果:
    1. 加上SG-DMA技术, 4次数据拷贝 -> 2次数据拷贝
    2. sendfile代替read + write, 4次上下文切换 -> 2次上下文切换(内核态<=>用户态)
    3. 整个过程中, CPU都没有参与数据拷贝的过程, 因为数据拷贝都交给DMA完成, 所以完全实现了零拷贝;
    4. 使用零拷贝可以把文件传输性能提高一倍以上, 但是零拷贝技术不能对文件内容进一步加工(例如压缩后再发送)

Pagecache有什么用?

这里的Pagecache也就对应上面图片里面从磁盘文件里面拷贝出来的缓冲区

  1. 其实就是内存中的高速缓冲区，因为读取磁盘的速度远远比不上读取内存的速度，所以把数据先保存到pagecache可以提升数据的读取速度

  2. 因为pagecache容量对比磁盘来说小很多，所以pagecache会淘汰一段时间内没用的数据

  3. 传输大文件的时候pagecache不适用，因为内存大小有限，而且大文件不是每部分都有用 -> 占用了热点小文件的空间 -> 降低了并发的处理效率

  4. 大文件拷贝不应该占用pagecache, 而零拷贝基于pagecache来实现, 所以, 大文件拷贝不适用零拷贝

  5. 大文件传输应该用什么实现？ 首先从传统的文件传输过程来切入

    传统的文件传输
    系统调用read函数
    内核向磁盘发起IO请求
    磁盘将数据准备好并向内核发起IO中断
    内核将数据从磁盘控制缓冲区读取到pagecache中
    内核将数据从pagecache拷贝到用户缓冲区
    read调用正常返回

  6. 上述过程存在的两个问题: 大文件占用了pagecache + read调用过程中阻塞进程

  7. 大文件占用了pagecache

    1. 内核直接将数据从磁盘控制器缓冲区拷贝到用户缓冲区, 跳过了pagecache
    2. 绕开pagecache的I/O叫做直接I/O; 使用pagecache的I/O叫做缓存I/O

  8. read调用过程中阻塞进程 -> 使用异步IO来解决, 用户进程发起异步IO读, 内核向磁盘发起读请求, 此时用户进程直接返回执行其他任务

  9. 所以, 针对大文件传输可以使用直接IO + 异步IO的方案来代替零拷贝技术; 对于Nginx来说, 判断文件是否为大文件的依据是: file_size > directio = 1024m, 也就是1G(仅供参考)

记录元素是否出现过

两个数组的交集

  1. 思路：利用两个unordered_set，一个用来去重、一个用来去重+查找操作O(1)，最后结果转成vector
  2. 这题不需要排序、不能让数据重复 -> 用unordered_set合适（底层实现是哈希表，读写效率高）
  3. 范围for循环和迭代器可以防止遍历越界

快乐数

  1. 思路：实现快乐数的计算方法，使用unordered_set查找计算结果是否出现过，如果出现过要及时退出，避免程序死循环

两数之和

  1. 思路：使用unordered_map存储已遍历过的键值对，遍历数组nums[i]，并查询target-nums[i]是否出现在umap中，如果出现，说明找到了和为target的两个元素
  2. 边找边加入更方便，不用考虑不同索引的元素相同的情况
  3. {}也是数组，结果可以直接返回花括号
  4. 关于哈希容器的选择
    1. 因为要返回索引，所以要存储键值对，一定要在3种map里面选一种
    2. 统计两数之和，只需要关注target-nums[i]和nums[i]，所以顺序不重要，为了获得更好的查找效率，选择unordered_map是最合适的

记录元素出现的次数

有效的字母异位词

  1. 思路：数组做哈希表；加法统计s各字母出现的次数、减法统计t各字母出现的次数，如果最后数组元素不全为0说明构成st的字母不相同

赎金信

  1. 思路：加法统计magazine里面字母出现的个数，减法统计ransomNote里面字母出现的个数，如果最终结果出现负数说明magazine无法构成ransomNote
  2. 固定大小的哈希表可以选用数组，数组相对于map来说不用维护红黑树不用做哈希映射，更简单直接
  3. 可以将遍历数组和遍历ransomNote结合起来，因为只要出现了一个负数，程序就可以返回结果了

四数相加Ⅱ

  1. 思路：数组之间是固定的，所以可以把ABCD四数相加拆成两两相加比较结果(A+B=-C-D)，最终看等号两边匹配成功的次数
  2. 哈希表的选择：因为find只会找一次，如果使用multiset的话虽然可以记录重复的两元素和，但是find只会匹配一次，后面就算有重复的值也不会被记录；所以这题需要用unordered_map去记录元素和+出现次数；使用find一次匹配就把组合数给统计完全
  3. 取负数的逻辑可以用0-，看起来比x(-1)更直观

修改字符串

反转字符串

  1. 思路：swap函数+双指针

反转字符串Ⅱ

  1. 思路：指针每次的步长设置为2 * k就可以，每次判断i + k是否满足条件，i + k <= s.size()的时候翻转k个字符，i + k > s.size()的时候翻转到最后一个字符
  2. 步长设置为i += 2*k的话（if语句的判断逻辑也是），可以让程序更直观，但是要注意reverse的区间是左闭右开的，也就是说右边界是取不到的，所以实际reverse的时候记得-1
  3. if 语句处理剩余位数够翻转k位的情况，剩下语句处理不足k位的情况

替换数字

  1. 思路：统计有多少个数字，扩容后从后往前遍历（不用移动后继元素）
  2. 两个并列的循环，复杂度是O(n) + O(n) = O(n)，嵌套也不一定就是O(n^2)
  3. 申请一个新的数组写法
    1. 碰到数字替换成number，注意索引让for循环控制就好了
    2. 如果需要自己控制expression中的变量，改变这个变量以后一定要检查
  4. 复用原字符数组的写法
    1. 先扩容，这题是计算有多少个数字，并根据数字的个数进行扩容
    2. 从后往前遍历并替换，从后往前替换不需要把后继元素后移

翻转字符串中的单词

  1. 思路：去除多余空格后，先翻转整个字符串，然后用空格分段翻转单词
  2. 去除多余空格的时候， 不要考虑前后，就考虑当前位置的元素是空格还是非空格，非空格就一次性加入
  3. 翻转单个单词的时候，翻转右边界是空格前，翻转左边界是空格后

右旋字符串

  1. 思路：先整体翻转，再根据给定的k值为边界分成两个子串，翻转两个子串
  2. 如果按照k来切割，要注意区间是左闭右开的[0, k)和[k, s.size())
  3. 所以实际翻转的时候区间是[0, k-1]和[k, s.size()-1]

字符串与模式串的匹配

实现strStr()

  1. 思路：使用KMP算法构造next数组匹配模式串，匹配成功就用当前元素索引-模式串长度得到开始位置索引
  2. KMP算法，找到最长相等前后缀，直接跳转到前面几位字母和当前元素相同的元素，避免从头遍历
  3. 关于next数组
    1. 记录next数组，next数组记录的是截至到索引为i的子串最长相等前后缀长度，也是要跳转元素的下标
    2. next数组最大的作用就是，当模式串与文本串不匹配的时候，回退到最近可重新匹配的位置
    3. 前缀就是不包含最后一个字符，所有以第一个字符为开始的连续子串；后缀就是不包含第一个字符，所有以最后一个字符结尾的连续子串
  4. KMP算法的意义在于将模式串的匹配问题从暴力解法的O(nxm)变成了O(n+m)，这里m指的是构造next数组的花费，n是实际匹配的花费

重复的子字符串

  1. 思路：KMP算法，最长相等前后缀中不包含的子串长度如果能够被整个字符串的长度整除，这个子串就能够重复构成整个字符串
  2. 思路Ⅱ：将字符串重复拼接起来，去掉首尾元素能够在中间发现原字符串说明原字符串是由某个子串构成的

数组查找

二分查找

  1. 思路：有序数组每次找中间点，大了在左区间继续找，小了在右区间继续找，必须分离开不满足条件的点
  2. 为什么这里一定要<=？ - 不一定，但是统一选<=
    1. <= 是定义了一个左闭右闭区间；坚持左闭右闭，最后可以缩小成一个点，例如，[1, 1]，right是可以取到的，所以要把不满足条件的点分离开不加入下一次查找（right = mid - 1）
    2. < 是定义了一个左闭右开区间，如果是左闭右开<，right是取不到的（right = mid），例如，[1, 1)这是个有问题的区间，所以right要是取不到的点，例如，[1, 2)
  3. 二分查找的条件是有序+不重复数组

长度最小的子数组

  1. 思路：滑动窗口内统计元素之和，右边界加入元素，左边界弹出元素（弹出之前要统计最小窗口值和重新计算窗口元素和）
  2. 缩小窗口的过程必须写成while循环，因为窗口内的总和大于target可以继续缩小
  3. 注意这题是找子数组，所以不能排序

区间和

  1. 思路：输入的时候就去求前缀和，并形成新的数组，要输出区间和就用右区间前缀和-左边界之前区间前缀和，时间复杂度是O(n+q)，q是查询次数
  2. 如果每次都对区间进行重复求和，时间复杂度是O(n*m)m，m是查询次数（会超时）
  3. scanf、printf性能更高，可格式化输入输出，适用于大规模数据读取输出操作；cin、cout是标准库函数，更直观、支持操作符>>和<<重载，适用于小规模数据读取输出操作

数组修改

移除元素

  1. 思路：快慢指针，快指针为目标值的时候直接自增，为非目标值的时候赋值给慢指针然后移动慢指针
  2. 快慢指针可以把一个数组当成两个来用，因为慢指针永远追不上快指针，最后的结果返回慢指针就行
  3. 数组删除元素本身就不是真的删除，而是调整自己有效区间的大小，值还在那个内存里，只是不属于数组了

有序数组的平方

  1. 思路：快慢指针，快指针遍历，慢指针计算平方；记得平方后元素顺序要调整
  2. 排序可以sort，这里复习一下快排

螺旋矩阵Ⅱ

  1. 思路：从1-n^2构成螺旋矩阵总共需要n / 2圈，每次都选择左闭右开区间，顺时针转动；如果n是奇数则额外处理最中心的位置
  2. 最外层要有loop来控制圈数，内层要有4个for循环来控制旋转的4个方向
  3. 起始位置和偏移量都是基于矩阵位置来计算的，考察对元素位置的访问
  4. 选好了填充元素的区间（比如左闭右开）4个方向都要遵循左闭右开，中途更换很容易乱

栈

用栈实现队列

  1. 思路：两个队列一个负责进一个负责出，元素入栈2次顺序就能变成队列的顺序
  2. 负责出的栈空了才去检查负责入的栈

有效的括号

  1. 思路：用栈来匹配最近的括号对，因为左右括号不相同所以需要构造映射关系
  2. 映射关系可以用关联容器来构造，或者直接在入栈的时候就推另一半括号（映射），更简洁

删除字符串中的所有相邻元素

  1. 思路：用栈来匹配相邻重复项，每次入栈之前都检查栈顶元素是否相同
  2. 更简洁的思路是直接把string当作一个栈

逆波兰表达式求值（每次碰到运算符拿栈）

  1. 思路：每次碰到运算符就取栈顶元素参与计算，将计算结果入栈
  2. 这里最好维护元素为int（int不够直接换long long）的栈，否则每次计算的时候都要转类型

队列

用队列实现栈

  1. 思路：定义一个输入输出队列一个临时队列，每次push元素时都先把所有前驱元素暂存临时队列再取回
  2. 也可以在pop或者top操作里面做上述操作
  3. 一个队列也可以实现，先push元素，然后再把所有前驱元素重新入队变成后继元素

滑动窗口最大值

  1. 思路：实现一个单调队列，保证队列元素是从大到小的（入队之前弹出所有比自己小的元素）；当窗口移动时删除元素先比对i-k是否为队首元素索引，满足条件才pop，否则不操作队列
  2. 滑动窗口用for循环来实现，单调队列直接用一个新类来实现，两者分开不要一起做
  3. deque是双端队列，有pop_back() 和 pop_front()两种弹出元素的方法

前K个高频元素

  1. 思路：排序以后用关联容器统计<元素，出现次数>，取出出现次数最大的前k个元素
  2. 统计出现次数最大，可以用sort传谓词；或者构造小顶堆（优先级队列），因为要舍弃出现次数最小元素
  3. 小顶堆的大小正好是k

二叉树的遍历(非递归遍历)

前序遍历

  1. 要注意用栈才能实现纵向遍历，用队列是横向的层序遍历
  2. 子节点入栈的顺序是右->左才能保证出栈的时候是先访问左孩子再访问右孩子

后序遍历

  1. 基于前序遍历来改，子节点入栈顺序是左->右，最后的result顺序是根->右->左
  2. 把result使用stl库函数reverse传两个迭代器进去就是后序遍历

中序遍历

  1. 访问完左孩子节点才能访问根节点和右孩子节点
  2. 前序遍历要访问的节点和要处理的节点是一样的，
  3. 中序遍历不一样，需要定义一个指针访问元素，再通过栈去处理元素

二叉树递归遍历和非递归遍历的区别

  1. 迭代遍历和递归遍历的时间复杂度都是O(n), 空间复杂度O(h)，h是树的高度
  2. 递归遍历深度受到系统调用栈的影响，迭代遍历使用的栈是用户自定义的，更灵活一些

二叉树的层序遍历（广度优先搜索）

  1. 队列每次存储一整层的元素，在弹出元素之前要先记录队列长度将一层元素通过for循环处理完毕
  2. 元素处理的过程是在for循环里面的
  3. 在循环体里面定义path，就能够隔离每一层元素
  4. N叉树的层序遍历，遍历孩子节点可以用迭代器来防止越界，另外，迭代器记得解引用
  5. 二叉树深度和高度的区别，深度是自顶向下，高度是自底向上
    1. 节点深度是根节点到该节点路径长度
    2. 节点高度该节点到最远叶子节点的路径长度
    3. 一整棵树的深度和高度是同一个

二叉树的遍历

  1. 当作普通二叉树来做 -> 层序遍历统计每一层的宽度
  2. 当作完全二叉树来做（不会也没关系，主要是统计完全二叉树的节点）
    1. 如果当前是满二叉树 -> (2 << depth) + 1
    2. 如果当前不是满二叉树 -> 递归遍历左右孩子节点直到碰到满二叉树（最后一个节点就是一个满二叉树）；countNode(leftDepth) + countNode(rightDepth) + 1最后这个加一是算根节点的
    3. 对于满二叉树来说，第k层有2^k-1个节点（k从1开始计算）

平衡二叉树

  1. 以当前节点为根节点的二叉树已经不是平衡二叉树，返回高度就没有意义了；这个逻辑没有处理好
  2. 记录节点高度的逻辑
    1. 空节点的高度是0
    2. 左右孩子有一个存在，那么对应的值+1；这里很巧的地方是1+max(lhs, rhs)的1+直接跳过了一个变量的定义，还记录了节点的高度
    3. 左右子树高度差超过1，直接返回-1
  3. 深度是自顶向下递增的，所以用前序遍历；高度是自底向上递增的，所以用后序遍历

二叉搜索树相关

二叉搜索树中的搜索

  1. 可以用非递归遍历（前序），碰到值相同的节点直接返回就好了
  2. 这题最好是用二叉搜索树的特性，左子树小右子树大，不需要额外定义一个栈来遍历BST

验证二叉搜索树

  1. 不能单纯判断根节点大于左子树和右子树，因为没有办法比较与叔伯节点之间的关系
  2. 二叉搜索树的中序遍历是个严格递增的有序数组，找中序遍历序列然后再验证

二叉搜索树的最小绝对差

  1. 二叉搜索树的中序遍历是个严格递增的有序数组，针对有序数组找最小绝对差就很简单了
  2. 也可以直接在中序遍历的过程中就找出节点的最小绝对差，因为要记录差所以需要一个容器装结果，一个指针指向前一个节点
  3. 递增数组的差天然就是绝对差，所以这题没必要加abs去求绝对值

二叉搜索树中的众数

  1. 要分离好第一个节点的情况、节点值相同的情况、节点值不相同的情况
  2. 如果出现频率和最大值相同直接push_back，出现频率大于最大值要先clear再push_back；不能合并

二叉搜索树的最近公共祖先

  1. 思路：二叉搜索树的最近公共祖先节点的值介于两者之间，这里不区分pq谁大谁小
  2. 遍历BST过程中第一个满足这个条件的节点就是最近的公共祖先
  3. 为了找到第一个值介于pq两者之间的节点，递归或者迭代法都可以实现

二叉搜索树中的插入操作

  1. 思路：pre，cur遍历二叉搜索树，利用 pre节点来接住新构造的节点newNode
  2. 需要有一个pre节点来接住新构造的节点newNode
  3. pre一开始就要拿到当前节点cur的值，否则cur往下遍历是有可能变成空指针的，pre变成空指针就没有办法变成新节点的父节点
  4. 向二叉搜索树插入元素只需要找到空节点就可以了，不需要调整二叉树的结构
  5. 递归法不是太熟

删除二叉搜索树中的节点

  1. 思路：判断与孩子节点之间的关系，存在以下五种情况
    1. 左右孩子都为空 -> 直接删除
    2. 左孩子为空 -> 右孩子代替当前节点
      1. 这里必须判断左孩子为空的情况，因为两边都非空的情况是在最后处理的
    3. 右孩子为空 -> 左孩子代替当前节点
      1. 这里必须判断右孩子为空的情况，因为两边都非空的情况是在最后处理的
    4. 左右孩子都不为空 -> 左子树接到右子树的最左下节点处后, 右孩子代替当前节点
    5. 没找到要删除的节点, 直接返回
  2. 不用往上判断，只需要判断当前节点是否为要删除的节点
  3. 把左子树接到右子树最左下节点的时候，判断的是cur->left孩子节点，因为要改变指针

修剪二叉搜索树

  1. 思路：判断当前节点，递归寻找满足条件的左右子树
    1. 如果值小于最小值的时候，返回当前节点的右子树（因为左子树只会更小
    2. 如果值大于最大值的时候，返回当前节点的左子树
    3. 最后要记得用root->left接左子树、root->right接右子树
  2. 为什么这个方法不需要在意子树的修剪？ 其实已经做了修剪的工作，返回值就是修剪后的树

将有序数组转换为二叉搜索树

  1. 递归遍历，拆分成左右两份数组+每次只构造根节点
  2. 注意左右两边数组的判空

将二叉搜索树转换为累加树

  1. 这题的关键在于需要按右->中->左的顺序去遍历二叉树
  2. 不属于三种深度优先遍历
  3. 需要一个指针来走向最右边节点，需要一个指针来记录上一个节点的值
  4. 当前节点只需要加上一个节点的值即可，每次只做一个操作

  1. prim三部曲

    2. 用visited数组来标记新加入的节点
    3. 更新加入节点后的生成树中最近节点距离其他非生成树节点的距离
  2. minDist里面记录的是索引代表节点离生成树的最小距离

  3. 要求两个点不在一个集合可以通过并查集来检查，使用并查集记得init();

  1. 和prim算法的步骤是相同的
  2. 不同之处在于，minDist数组是用来记录与源节点的最短距离的，所以遍历完所有顶点后只需要输出尾元素
  3. 堆优化中，堆的作用有两个，
    1. 存储距离源点最近顶点的编号
    2. 排序，让距离源点最近的顶点在top的位置，可以直接取出不用每次都进行条件判断
    3. 因为是有堆来组织元素，所以不需要visited数组去管理元素是否有遍历到，堆用完直接pop
    4. 这里lhs.second > rhs.second的判断逻辑是因为，原始的priority_queue是个基于<运算的大顶堆，我们需要的是小的元素在上面，所以需要的是小顶堆，于是在这里对大小判断作一个取反操作

Bellman_ford算法（边视角、边视角、边视角）

  1. 什么是松弛？ 松弛就是更新from到to的最短距离，涉及到“更新”这个概念是因为每次松弛，能够使用的边数量都不一样
  2. 松弛每次都要保证已经visited过的节点与源点的距离是最近的，要求n到源点最近距离，因为n个顶点有n-1条边，所以当所有边都加进来的时候minDist就应该存储n到1的最近距离，所以只需要松弛n-1次
  3. 不需要visited数组，只需要判断from能不能到达
  4. 队列优化版本Bellman_ford：SPFA算法（边视角）用队列优化的地方在于
    1. 只基于上一次松弛过的节点进行松弛，也就是把上一次松弛过的点作为新的出发点去松弛相连的边
    2. 图越稀疏效率越高，如果是稠密图SPFA算法效率接近普通版本的bellman_ford
  5. 单源有限最短回路：minDist_copy[from] != INT_MAX没有办法处理带负权回路的情况，所以当图中有负权回路并且上一个from节点我们遍历过的时候会基于本次松弛的minDist多一次松弛
    1. 面对有负权回路的情况下，如果我们要控制松弛次数，需要拷贝上一次松弛的结果
    2. 然后基于上一次松弛的结果来做本次松弛

  1. 使用变量的建议
  2. 数值超过了int的表示范围直接用long long
  3. 执行浮点数运算用double
  4. 带符号数designed会自动转换成无符号数undesigned
  5. 对象object就是一片能够存储数据并且具有类型的内存空间
  6. 在变量名字前加extern可以只声明变量, 不定义变量 -> 避免其他文件对该变量的重复定义
  7. void* 可用于存放任意对象的地址, 是一种特殊的指针类型；不能对void*执行解引用操作
  8. const对象一定要初始化；const对象仅限在当前文件里面使用，如果想要让const对象跨文件使用可以在声明和定义的时候在const前面加extern
  9. 常量表达式constexpr就是在编译过程中就能得到计算结果的表达式，自己定义的类不属于字面值类（字面值就是不能改变的值）不能被定义成constexptr
  10. auto会忽略顶层const，保留底层const
  11. decltype()的作用是返回括号内的表达式计算结果对应的类型；如果在括号里面加括号，返回的永远是引用

第三章 - 字符串、向量和数组

  1. 头文件不应该包含using声明，因为每个使用了该头文件的文件都会具有这个声明，有可能包含到我们不想要的名字，产生难以预料的错误
  2. 用cin读取，碰到空白就会停下；如果想要读取一整行数据可以用getline(&istream, string)
  3. 迭代器指向某个元素，或者指向尾元素的下一个位置；迭代器的end成员就是返回尾元素的下一位置，也叫做尾后迭代器

第四章 - 表达式

  1. 左值代表对象的身份，右值代表对象的内容（值）；右值可以用左值来代替，左值不能用右值代替
  2. 赋值运算符满足右结合律，也就是说先算后面的等号
  3. 为什么少用i++，因为是返回自增之前的值，系统会为原始值保留副本，也就是说比++i多了一次拷贝
  4. sizeof()返回一个表达式或者一个类型名字所占用的字节数，对数组执行sizeof函数可以求出数组所占空间大小
  5. 强制类型转换cast，不带底层const的可以用static_cast<要转换类型>；const_cast<要转换类型>只能改变运算对象的底层const

第六章 - 函数

  1. 自动对象就是在花括号里面创建的对象，一旦出了花括号这个对象就被销毁了，值会变成未定义的
  2. static类型的生命周期能够贯穿函数调用以及之后的时间，一直到程序终止才被销毁
  3. 一个函数里面定义的变量叫做局部变量，会隐藏外层作用域中同名的变量
  4. 实参初始化形参的时候会忽略掉顶层const，所以尽量多使用常量引用，来限制函数所接受的实参
  5. initializer_list这个标准库类型允许我们定义不同数量的实参，但是类型一定都是type；…也可以，但是只能放在形参列表的最后一个位置
  6. 函数不能返回数组，但是可以返回指向数组的指针或者是数组的引用、定义类型别名；也可以用尾置返回类型返回一个数组auto func(int i) -> int (*) [10];
  7. 重载函数就是函数名相同，形参列表不同的一组函数；举例：只有返回类型相同是不行的；但是加不加const是一对重载函数
  8. inline和constexpr通常定义在头文件内
  9. inline可避免函数调用的开销，在调用点上‘内联地’展开；用于优化规模小、流程直接、调用频繁的函数
  10. constexpr返回类型和所有形参都必须是字面值类型，被隐式地归为内联函数
  11. assert是一种预处理宏，用来检查不能发生的条件，assert(expr)会先计算expr的结果
  12. 如果表达式为假（0），assert会输出信息并终止程序执行；
  13. 如果表达式为真（非0），assert什么也不做；
  14. assert定义在cassert头文件中
  15. assert行为依赖于NDEBUG这个预处理变量，如果没有定义NDEBUG，会正常进行检查并且执行#ifndef和#endif之间的代码；如果定义了NDEBUG，不会检查，也就是关闭了调试状态
  16. 函数指针指向的是函数而不是一个对象，函数的类型由返回类型和形参类型共同决定；如果想要定义指向某个函数的函数指针，只需要把函数名字换成(*指针名字)就可以
  17. 函数重载就是一个名字可以定义多个函数，根据现场有无声明以及匹配程度来选择可用函数；重载函数的形参数量或者形参类型不同，目的是为了避免二义性调用，也就是找不到唯一最佳匹配
  18. 候选函数 -> 可行函数（形参数量=实参数量、实参类型能转换成形参类型） -> 最佳匹配
